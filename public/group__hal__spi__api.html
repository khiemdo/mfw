s_hal_spi_ctx_t* hal_spi_init ( E_HAL_SPI_PORT_t  e_port,
E_HAL_SPI_MODE_t  e_mode,
uint32_t  ui32_spiClock 
)

Initializes a SPI port of the microcontroller.

A call to this function it is required prior to any SPI operations!

Attention
This function does not initialize any chip select pins. It is up to the user to initialize and handle the chip select signals to the connected SPI slave(s) before starting the SPI transfer. Furthermore, this SPI driver uses DMA transfers and thereby blocks the corresponding DMA channels for the SPI ports (see E_HAL_SPI_PORT_t). Make sure the DMA controller is initialized before calling this function.
Parameters
e_portSPI port to be initiliazed.
e_modeSPI mode to be used.
ui32_spiClockSPI clock speed to be used in Hz.
Returns
Returns pointer to context variable if SPI port is initialized successfully. NULL otherwise.

void hal_spi_xfer ( s_hal_spi_ctx_t *  ps_ctx,
uint8_t *  pc_txData,
uint8_t *  pc_rxData,
uint16_t  ui16_len,
pfn_spi_callback  pf_cb 
)

Starts a SPI transfer in a non-blocking manner.

Prior to call to this function it is required initialize the SPI port first! The context pointer returned by hal_spi_init must be given as parameter here. This function starts a non-blocking SPI transfer, that is, the function will return to the caller right after initial processing. The specified callback function is called when the SPI transfer is completed.

Attention
This function does not handle any chip select pins. It is up to the user to initialize and handle the chip select signals to the connected SPI slave(s) before starting the SPI transfer.
Parameters
ps_ctxPointer to the SPI port context (previously returned by hal_spi_init).
pc_txDataOptional: Pointer to the data to be transmitted. This value might be NULL if no data shall be transmitted but only data reception is required. If set to NULL the SPI driver will transmit dummy bytes instead to allow data reception from the SPI slave.
pc_rxDataOptional: Pointer to memory to store received data. This value might be NULL if no data shall be received but only data transmission is required. If set to NULL the SPI driver will discard any received bytes.
ui16_lenLength of the given data memories. Make sure the number matches the size of TX and RX data memory if both options are used.
pf_cbFunction pointer that is called when SPI transfer is completed.
Returns
None.

Starts a SPI transfer in a blocking manner.

Prior to call to this function it is required initialize the SPI port first! The context pointer returned by hal_spi_init must be given as parameter here. This function starts a blocking SPI transfer, that is, the function will not return immediately but instead returns when the SPI transfer is completed. This function serves as counter part to the non-blocking function hal_spi_xfer.

Attention
This function does not handle any chip select pins. It is up to the user to initialize and handle the chip select signals to the connected SPI slave(s) before starting the SPI transfer.
